Detailed Description

template<typename KeyType, typename ValueType>
class pluginplay::cache::database::RocksDB< KeyType, ValueType >

Wraps RocksDB in PluginPlay's database API.

This DatabaseAPI can be used to implement a Database whose data is managed by RocksDB. The RocksDB class has its own PIMPL which wraps the calls to RocksDB, and handles dispatch if the user opted not to build PluginPlay with RocksDB support. In the case that PluginPlay was built without RocksDB support attempting to use this class will result in runtime errors.

N.B. API documentation assumes that RocksDB support has been enabled. If it has not, then attempting to call any method other than the default ctor or the dtor will result in a runtime error.

Template Parameters
KeyTypeType of the keys in the database, expected to be some type which holds binary data.
ValueTypeType of the values that the keys map to. Exected to be a type holding binary data.

immutable reference to a value

Because different backends have different memory semantics, insulating the user from these semantics requires a more complex solution than simply using const ValueType&. To this end we have created the ConstValue class. The ConstValue class is capable of holding either a const reference to an object or the value itself. Calling .get() on the ConstValue instance returns an immutable reference to the wrapped value, regardless of whether the ConstValue instance owns the mapped value or if the backend does. This type is an alias of ConstValue<ValueType>.

Creates a, or opens an existing, RocksDB database.

This Ctor is used to open an existing database (if path already exists and is a RocksDB database) or to create a new database (if path does not point to an existing file). Errors will arise if the RockSDB backend can not open the database (path already exists), or if the backend can not create the database. If this call is sucessful then the database is ready for business.

Parameters
[in]pathWhere the RocksDB database lives/will live. If path is an already existing RocksDB database the resulting instance will open it. If path is not an existing database then a new database will be created and opend.
Exceptions
std::bad_allocif the PIMPL can not be created. Strong throw guarantee.
