Detailed Description

Implements the RocksDB class when RocksDB support is enabled.

When the user has requested RocksDB support, rocksdb.cpp will select this header file. As defined in this header file (and the corresponding *.ipp file), the RocksDBPIMPL wraps a RocksDB database with a minimal API so that it can implement the RocksDB class. All of the actual RocksDB calls, types, etc. are in this class.

N.B. RocksDB only supports values that are about 4GB in size. Since the data is stored in the database as a string we can circumvent this problem by splitting the value into smaller chunks. The splitting and reassembling of values happens automatically and users of this database should act as if it doesn't happen.

explicit

Creates (or opens) a RocksDB database with the specified path.

This Ctor is used to open an existing database (if path already exists and is a RocksDB database) or to create a new database (if path does not point to an existing file). Errors will arise if the RockSDB backend can not open the database (path already exists), or if the backend can not create the database. If this call is sucessful then the database is ready for business.

At the moment, construction uses the default RocksDB options for the database. No tuning has been performed.

Parameters
[in]pathFor new databases this is where the database should live, for existing databases this is where it lives.
Exceptions
NoneNo throw guarantee. At the moment, if an error occurs an assertion is tripped.
