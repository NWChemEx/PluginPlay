Detailed Description

template<typename KeyType>
class pluginplay::cache::UUIDMapper< KeyType >

Object responsible for assigning UUIDs to objects (and remembering them)

The UUIDMapper is responsible for assigning UUIDs to objects and maintaining a record of this mapping. Conceptually this means that UUIDMapper is viewable as a DatabasePIMPL<KeyType, boost::uuids::uuid> instance.

Template Parameters
KeyTypeThe type of the objects having UUIDs assigned to them.

◆ insert()

template<typename KeyType >
void pluginplay::cache::UUIDMapper< KeyType >::insert ( key_type  key)

Overloads insert so that the user doesn't need to provide a UUID.

The insert method of the base class takes both a key and a value. That method is useful for reloading a saved state into a UUIDMapper instance. For initially generating the state this overload is more useful. This overload calls the internal uuid_ function (which in turn creates a new UUID) and forwards the key, and the newly created UUID, to the base class's insert method.

N.B. To help avoid accidentally overwriting a UUID this function is a no-op if count(key) is true. If you really want to overwrite the UUID call free(key) first.

Parameters
[in]keyThe object getting a UUID assigned to it. If key already has a UUID this is a no-op.
Exceptions
boost::uuids::entropy_errorif there is a problem generating the UUID due to insufficient entropy. Strong throw guarantee.
???If the wrapped database's insert method throws. Same throw gurantee.
